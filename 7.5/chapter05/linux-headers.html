5.6. Linux-3.13.3 API 头文件

The Linux API Headers (in linux-3.13.3.tar.xz) expose the kernel's API for use by Glibc.

预计构建时间: 0.1 SBU
磁盘空间需求: 584 MB

5.6.1. 安装 Linux API 头文件

Linux 内核需要暴露应用程序接口（API）给系统的 C 库（LFS 中是 Glibc）使用。这通过清洁 Linux 源码包中的各种头文件来完成。

要确保不存在以前产生的文件和依赖： 

make mrproper

现在从源代码中测试并提取用户可见的内核头文件。它们被放在另外的一个目录中，然后被复制到需要的位置，因为提取过程会移除目标目录中的所有文件。 

make headers_check
make INSTALL_HDR_PATH=dest headers_install
cp -rv dest/include/* /tools/include

这个包的详细信息位于 第 6.7.2 节 “Contents of Linux API Headers.”
